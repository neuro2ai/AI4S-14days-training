# 2. 我仍然不懂什么

1. 随机梯度下降的原理是什么？  

原理： 梯度下降的核心是寻找函数的“最小值”（即 Loss 最低点）。  

• “梯度”：函数在某一点的导数，指向数值上升最快的方向。  
• “下降”：我们要沿着梯度的反方向走，这样才能让 Loss 减小。  
• “随机”：在传统的梯度下降中，每走一步都要计算全量数据（比如 100 万个细胞），速度极慢。而在 SGD中，我们每次只随机抽取一小部分数据（一个 Batch，比如 2 个细胞）来计算梯度并更新参数。  

优势： 速度快，且这种“随机性”带来的波动有时能帮助模型跳出局部最小值，找到更好的全局最优解  

2. model.parameters() 如何自动识别需要更新的参数？  

这是一个自动登记机制：  

继承关系：当你定义类时使用了 class CellEncoder(nn.Module)，你的模型就继承了 PyTorch 的管理系统。  

自动识别：在 init 中，凡是赋值给 self 的 PyTorch 标准层（如 nn.Linear），它们内部定义的 weight（权重）和 bias（偏置）都会被自动包装成 nn.Parameter 对象。  

登记入册：nn.Parameter 是一种特殊的张量，它默认开启了 requires_grad=True（需要计算梯度）。  

一键调用：当优化器调用 model.parameters() 时，PyTorch 会扫描模型中所有被标记为 nn.Parameter 的变量并返回。如果你自己定义了一个普通的变量（如 self.temp = 0.5），它不是 nn.Parameter，就不会被返回，也不会被优化  

3. 步长是什么，大小有什么影响？  

步长（在代码中即 lr）决定了模型根据错误方向“迈多大的步子”去修改参数。  

步长过大（High $lr$）：参数更新的步长超过了损失函数（Loss Function）局部极小值的覆盖范围，导致参数在目标点两侧剧烈震荡（Oscillation）甚至发散（Divergence）。表现为 Loss 数值不减反增，或因数值超出浮点数表示范围而产生 NaN（Not a Number）。  

步长过小（Low $lr$）：参数更新幅度极微，导致模型收敛速度极慢。模型容易陷入训练时间过长的问题，且更容易停留在损失平原或非全局最优的**局部极小值（Local Minima）或鞍点（Saddle Points）中。  

理想步长（Optimal $lr$）：能够在保证数值稳定性的前提下，使损失函数以最快的速率下降并最终收敛（Convergence）**到全局或接近全局的最优解。  

4. 梯度的默认累加有什么应用？  

• 普通训练：算 1 个小 Batch 的梯度 → 立刻更新一次参数（step）。  
• 梯度累积：算 N 个小 Batch 的梯度并加在一起 → 只在最后更新一次参数（step）。  

假设你想用 Batch Size = 100，但显存只能装下 10）：  

1. 你跑 10 次 Batch Size = 10 的运算。  
2. 每次算出梯度后，不执行 zero_grad()，让它们在参数的 .grad 属性里叠在一起。  
3. 这 10 次叠完后的总梯度，在数学上等同于你一次性跑 100 个样本算出的梯度。  
4. 关键点：此时你再执行 一次 optimizer.step()。因为你只更新了一次，所以参数并没有“飞起”，它只是像跑了一个大 Batch 一样稳健地迈出了一步。  

在处理单细胞数据或大蛋白质模型时，有些隐藏的特征（比如某个稀有细胞亚型）在 10 个样本的小 Batch 里可能全是噪声，只有在 1000 个样本的大 Batch 里，信号才会抵消掉随机波动，显现出真实的梯度方向。  

5. 今天第一次执行完的平均总得分是1.3499819835027058，优化一次之后得到的总得分是 1.4723061323165894，为什么反而变大了？  

（1）. 步长（Learning Rate）太大：跑过头了  

这是最主要的原因。  

• 现象：模型发现了一个报错的方向，试图去改。但因为你的 lr 设得太高（比如 0.1 或 0.01 对某些初始化来说太大了），模型“用力过猛”，直接跨过了 Loss 最低的那个谷底，冲到了对面更高的山坡上。  

• 结果：参数动了，但改错的方向是对的，迈的步子却太大了，导致 Loss 爆炸。  

（2）. 样本顺序的随机性（如果你用了 shuffle=True）  

• 现象：第一轮（Epoch 1）计算的是模型对这 5 个样本的平均评价。第二轮优化后，如果 DataLoader 重新打乱了样本顺序，或者你只看了一个 Batch 的结果。  

• 逻辑：模型可能在某些样本上进步了，但在另一些还没见过的样本组合上表现得更差。在训练的前几轮，这种局部波动非常正常。  

（3）. 初始化的“阵痛期”  

• 现象：模型刚开始是随机初始化的。第一步优化就像是在完全黑暗的山谷里盲人摸象。  

• 逻辑：第一步迈出去，模型可能正好撞上了一个陡坡。此时的梯度信号还不稳定，Loss 可能会短暂上升。  

总之，在随机初始化、小数据集、shuffle=True、只训练很少步数的情况下，总 loss 一次更新后变大是正常现象，不能单凭一次变大就断言学习率过大。

# 3. 今天最重要的概念

1. optimizer = torch.optim.SGD(model.parameters(), lr = 0.01)：SGD是随机梯度下降，最经典的算法，稳，但有时改错的速度比较慢，也可以换成Adam（最主流最聪明的算法，会自动调整学习率）等优化器。model.parameters()是告诉优化器“你要管哪些参数”，不需要外部输入，任务是扫描模型内部，把需要更新的数字全部找出来给优化器。lr = 0.01是学习率，决定了每次根据梯度更新参数时的步长大小。  

| 优化器名称 | 缩写 | 核心特点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **随机梯度下降** | **SGD** | 基础、稳定。每一步的步长（学习率）固定。 | 简单任务，或需要精细调优的后期。 |
| **自适应梯度** | **Adagrad** | 给经常出现的特征分配较小的学习率，不常出现的给较大的。 | 处理稀疏数据（如果把单细胞输入看成高维且部分特征稀疏，它可以作为一种可类比的优化器思路）。 |
| **均方根传播** | **RMSprop** | 解决了 Adagrad 学习率下降过快的问题。 | 循环神经网络（RNN）或非平稳目标。 |
| **自适应矩估计** | **Adam** | **目前最通用**。结合了动量和自适应学习率，收敛飞快。 | **不知道选哪个时的首选**，AI4S 常客。 |

在数学中，线性层做的运算是 $y = xA^T + b$。  

• 输入x：形状是 (batch_size, 4)。  
• 输出 y：你想要的形状是 (batch_size, 8)。  
• 权重A：为了让 $x$ 的 4 列能和权重矩阵匹配，并映射出 8 个特征，权重矩阵的内部形状被设定为 (out_features, in_features)，即 (8, 4)。
