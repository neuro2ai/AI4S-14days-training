# 2. 我仍然不懂什么

1. 为什么每个细胞的目标标签是两位向量？因为CellEncoder 最后输出的 shape 是 (2, 2)，这意味着模型对每个样本会吐出两个数（比如 [0.8, 0.2]）。为了计算 Loss，标签必须也长成 [1.0, 0.0]，这样两个向量才能“面对面”比对误差。  

2. 这个标签怎么理解意义？  

[1, 0] 是目标向量，模型输出 [0.8, 0.2] 是当前预测向量；MSELoss 比较的是这两个向量之间的数值差异。  

只有在特定设置下，比如：最后一层接了 softmax或者你明确把输出解释成概率，它们才是概率。  

[1, 0] 的意思就是：我百分之百确定它是 A 类。模型输出 [0.8, 0.2]，0.8是模型觉得是 A 的概率。0.2是模型觉得是 B 的概率。[1, 0] 是终点，模型的输出[0.8, 0.2]是现在模型所站的位置，loss模型是用0.2的差距去优化参数，使下次结果更接近终点  

3. 除了criterion = nn.MSELoss()这种定义方法 ，还有什么其他定义方法  

| 损失函数名称 | 英文缩写 | 适用场景 | 朴素解释 |
| :--- | :--- | :--- | :--- |
| **均方误差** | **MSE** | 回归问题（预测直径、浓度） | 计算预测值与真值差的平方，对大误差罚分极狠。 |
| **平均绝对误差** | **L1** | 回归问题（数据有脏点时） | 只看差的绝对值，比 MSE 更“佛系”，不容易被离群点带偏。 |
| **交叉熵损失** | **CrossEntropy** | 分类问题（相分离预测） | 专门看概率分布。如果模型猜对了但不够自信，也会扣分。 |
| **二元交叉熵** | **BCE** | 二分类问题（是 / 否） | CrossEntropy 的简化版，专门处理 0 和 1 的选择。 |  

4. one-hot是什么，有什么优点，只分两类会不会比较简单，除此之外还有什么  

One-hot（独热）这个名字的由来：在长长的一串数字里，只有一个位置是“热”的（即 1），其他全是“冷”的（即 0）。这个顺序是自己定义的，第一种蛋白的索引是0，第一位的向量是1，索引为2的就把1依次往后移。所以即使只有0和1也可以表达定义多种种类，进行多分类  

• 第 1 种蛋白（比如 HSF1）：[1, 0, 0, 0, ... 0]（共 100 位）  
• 第 2 种蛋白（比如 Pol II）：[0, 1, 0, 0, ... 0]  
• 第 100 种蛋白：[0, 0, 0, 0, ... 1]  

优点：  

• 消除等级歧视：如果你用数字 0 和 1 标注，模型可能会潜意识觉得 1 比 0 大。用向量则表示两类在地位上是平行的。  

• 对接概率输出：模型的输出通常也是个向量（如 [0.8, 0.2]），One-hot 标注能让 Loss 函数直接计算两个向量之间的“分布距离”。  

• 支持多分类：如果你有 100 种蛋白，One-hot 依然能清晰地把它们区分开，不会产生数值混淆。  

| 方法名称 | 适用生物场景示例 | 标签形式示例 | 核心优点 |
| :--- | :--- | :--- | :--- |
| **二分类** | 预测 HSF1 是否发生相分离 | `[1, 0]` 或 `[0, 1]` | 简单明确，适合初步筛选。 |
| **多分类** | 区分 HSF1, Pol II, FUS 三种蛋白 | `[1, 0, 0]`, `[0, 1, 0]`, `[0, 0, 1]` | 类别之间地位平等，互不干扰。 |
| **多标签分类** | 细胞同时发生“相分离”和“磷酸化” | `[1, 1]` | 一个样本可以同时拥有多个属性。 |
| **回归（非分类）** | 预测液滴的精确直径 (μm) | `1.25` (具体的实数) | 能够提供定量的精确连续数值。 | 

5. 每个 batch 的 loss 是两个对象的“平均值”吗？  

是的，默认情况下确实是求平均。在 PyTorch 中，大多数损失函数（如 nn.MSELoss 或 nn.CrossEntropyLoss）都有一个参数叫 reduction，它的默认值是 'mean'（平均）。  

6. 如果说多次执行y = model(batch)，算是开始优化模型吗？  

没有，原因如下：  

• 参数没动：你的代码只是把数据塞进模型（Forward），算出模型有多错（Loss），然后把错误分打印出来。  

• Loss 波动是假象：如果你发现第二次运行的 Loss 和第一次不一样（比如从 1.38 变成了 0.98），那仅仅是因为 shuffle=True 让 DataLoader 每次抓取的样本顺序不同，或者是模型在不同批次上的表现差异，并不是模型变聪明了。

# 3.今天最重要的概念

1. One-Hot Encoding（独热编码）：用0和1代表类别A和B的关与开，它能与模型输出的 概率分布（如 [0.8, 0.2]）进行直接比对。通过计算这两者之间的交叉熵损失，模型可以量化预测值与真值之间的‘距离’，从而在反向传播中精准地调整参数，直到模型的判断与实验观察完全一致。  

2. criterion = nn.MSELoss(): 定义loss,MSE 是loss=(预测−真实)2  

3. 计算loss：  

(1) for batch_idx, batch in enumerate(loader): loader每次会调出一个batch，batch_idx是这个batch的编号，batch = 2则包含了序号为（0，1）/（2，3）等两个样品  

(2) y = model(batch)：把batch中细胞数据喂给模型，模型输出两个答案，存入y  

(3) t = target[batch_idx*2: batch_idx*2 + batch.shape[0]]：*2是因为batch_size是2，当 batch_idx = 0 时：取出 target[0:2]（第 0, 1 行答案）,当 batch_idx = 1 时：取出 target[2:4]（第 2, 3 行答案）。batch.shape[0] 是为了防止“最后一袋不满”。如果总共 5 个样本，最后一袋只有 1 个，这一行能保证只取 1 个答案，不会取多。0是因为在最后那一袋（只有 1 个样本时），batch.shape 变成了 (1, 4)。这时候 batch.shape[0] 就会自动变成 1  

4. loss = criterion(y, t): 判分，y：模型刚刚乱猜的 2 个答案（模型第一步是乱猜，因为model = CellEncoder这个模型的权重和偏置最开始是随机初始化的，后面才会根据loss去逐渐矫正）。t：你刚刚从大表里抠出来的 2 个标准答案。criterion：判分标准。它比对一下预测和标准，算出一个总的“错误分”。  

5. print(loss.item()): 看一眼分数，.item() 的作用是：把这个loss这个tensor（带有 grad_fn（梯度追踪器） 的张量，记得这个分数是由哪两个y和t算出来的，这两个y又是经过哪些W和b变换来的）变成一个普通的 Python 数字，方便你 print 出来看模型错得有多惨。item后面加个括号是因为loss.item 是方法本身，loss.item() 才是调用这个方法，得到一个普通数字  

6. print(loss):会返回grad_fn=<MseLossBackward0>  

当你后续执行 loss.backward() 时，PyTorch 会顺着这个 MseLossBackward 往回找：找到算这个 Loss 的 y。再通过 y 找到模型里的权重 $W$ 和 $b$。最后计算出每个 $W$ 应该负多少责（梯度）。  

7. 如果想要计算总的得分，代码如下：  

```python
total_loss = 0.0  # 1. 准备计分本
for batch_idx, batch in enumerate(loader):
    y = model(batch)
    t = target[batch_idx2 : batch_idx2 + batch.shape[0]]
    loss = criterion(y, t)
    # 2. 累加得分。注意一定要用 .item()，否则会拖慢内存！
    total_loss += loss.item()
    print(f"Batch {batch_idx} loss: {loss.item()}")
#3. 循环结束后，算出平均分
avg_loss = total_loss / len(loader)
print(f"--- 这一轮的平均总得分 (Average Epoch Loss): {avg_loss} ---")
```
